# Traceability Matrix

> **Версія:** v1.0 | **Дата:** 2026-04-22  
> **Джерело вимог:** [SSD](../architecture/SSD.md)  
> **Джерело стратегії тестування:** [Test Strategy](test-strategy.md)

Матриця гарантує, що кожна вимога з SSD покрита відповідними тест-кейсами та жоден функціонал не виходить у продакшн без перевірки.

---

## Матриця простежуваності

| Вимога (ID) | Опис вимоги | Тест-кейс (TC ID) | Тип тесту | Інструмент | Очікуваний результат |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **FR-01** | Реєстрація користувача | **TC-01.1** | API (Positive) | Postman | `201 Created`, запис у БД |
| **FR-01** | Реєстрація користувача | **TC-01.2** | API (Negative) | Postman | `409 Conflict` (email вже існує) |
| **FR-01** | Реєстрація користувача | **TC-01.3** | API (Negative) | Postman | `422 Unprocessable Entity` (невалідний email) |
| **FR-01** | Реєстрація користувача | **TC-01.4** | Unit | Jest/Pytest | bcrypt-хеш пароля збережено (не plain text) |
| **FR-02** | Авторизація (Вхід) | **TC-02.1** | API (Positive) | Postman | `200 OK`, отримання валідного JWT |
| **FR-02** | Авторизація (Вхід) | **TC-02.2** | API (Negative) | Postman | `401 Unauthorized` (невірний пароль) |
| **FR-02** | Авторизація (Вхід) | **TC-02.3** | API (Negative) | Postman | `401 Unauthorized` (неіснуючий email) |
| **FR-03** | Валідація / Refresh токена | **TC-03.1** | API (Positive) | Postman | `200 OK`, новий `accessToken` за валідним refresh |
| **FR-03** | Валідація / Refresh токена | **TC-03.2** | Security | Postman | `401 Unauthorized` (прострочений токен) |
| **FR-03** | Валідація / Refresh токена | **TC-03.3** | Security | Postman | `401 Unauthorized` (підроблений підпис) |
| **NFR-SEC** | Захист від Brute-force | **TC-04.1** | Security/Perf | JMeter | `429 Too Many Requests` після 5 спроб за 10 хв |
| **NFR-PERF** | Продуктивність ≥ 1000 req/min | **TC-05.1** | Performance | JMeter | p95 ≤ 150 мс, 0 помилок під навантаженням |
| **NFR-AVAIL** | Доступність 99.99% | **TC-06.1** | Smoke | Postman/Newman | Сервіс відповідає після рестарту пода |

## Правила оновлення матриці

1. При додаванні нової вимоги в SSD — додати відповідний рядок у матрицю до merge PR.
2. При видаленні вимоги — видалити або позначити пов'язані TC як `DEPRECATED`.
3. При зміні очікуваного результату — оновити колонку "Очікуваний результат" та версію матриці.
