# Test Strategy

> **Версія:** v1.1 | **Дата:** 2026-04-23  
> **Проєкт:** Auth Microservice REST API  
> **Залежить від:** [SSD — функціональні та нефункціональні вимоги](../architecture/SSD.md)

---

## 1. Загальний підхід

Стратегія базується на **API-first тестуванні**. UI-тестування не застосовується — сервіс не має графічного інтерфейсу. Автотести запускаються автоматично в CI/CD пайплайні при кожному Pull Request.

**Принцип Shift-Left:** виявлення дефектів якомога раніше — на рівні unit-тестів та code review, а не на продакшні.

---

## 2. Рівні тестування

### 2.1 Unit-тести

| Параметр | Значення |
| :--- | :--- |
| **Що тестується** | Бізнес-логіка Auth Service, JWT Manager (хешування, генерація/верифікація токенів) |
| **Відповідальні** | Розробники |
| **Інструменти** | Jest (Node.js) / Pytest (Python) |
| **Покриття (мін.)** | 80% рядків коду |
| **Тригер** | Кожен commit / PR |

### 2.2 Integration / API-тести

| Параметр | Значення |
| :--- | :--- |
| **Що тестується** | REST-ендпоінти: коректність статус-кодів, форматів відповідей, бізнес-правил |
| **Відповідальні** | QA-інженери |
| **Інструменти** | Postman + Newman (CLI-запуск у CI) |
| **Середовище** | Staging (ізольований Docker Compose) |
| **Тригер** | Кожен PR перед merge |

### 2.3 Security-тести

| Параметр | Значення |
| :--- | :--- |
| **Що тестується** | Brute-force захист, валідність/відкликання токенів, HTTPS-примус |
| **Відповідальні** | QA / Security-спеціалісти |
| **Інструменти** | Postman (negative cases), OWASP ZAP (DAST) |
| **Тригер** | Перед кожним релізом |

### 2.4 Performance-тести

| Параметр | Значення |
| :--- | :--- |
| **Що тестується** | Пропускна здатність (≥ 1 000 req/min), час відповіді (≤ 150 мс) — NFR-PERF |
| **Відповідальні** | QA / DevOps |
| **Інструменти** | Apache JMeter |
| **Тригер** | Щотижня та перед кожним релізом |

---

## 3. Тестові середовища

| Середовище | Призначення | Дані |
| :--- | :--- | :--- |
| **Local** | Розробка, debug | Синтетичні |
| **Staging** | Автоматизовані API/Security тести | Синтетичні |
| **Pre-prod** | Performance тести, smoke-тести | Анонімізовані |

---

## 4. Definition of Done

- Усі unit-тести проходять (`GREEN`).
- Покриття коду ≥ 80%.
- Усі API-тести в Postman-колекції проходять.
- Час відповіді p95 ≤ 150 мс під навантаженням 1 000 req/min.
- Жодних відкритих High/Critical вразливостей у SAST/DAST.
